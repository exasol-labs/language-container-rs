# Plan: add-slc-os-license-notices

## Summary

Ship an OS-layer third-party attribution bundle (manifest + canonical SPDX license texts + a written source offer) inside the distributed SLC tarball so the redistribution of Alpine apk packages and the bundled glibc/GCC runtime libraries is license-compliant, and bump the workspace version to 0.13.1.

## Design

### Context

The distributed SLC artifact is `lc-rs.tar.gz`, built by `Dockerfile.alpine` (`--target artifact`) and uploaded to BucketFS by `scripts/install.sh`. That tarball flattens the entire `alpine:3` rootfs **plus** the glibc/libstdc++ runtime libraries copied in via `cp -L` from the `rust:1.92-bookworm` (Debian 12) builder stage. The existing `THIRD-PARTY-LICENSES.md` is generated by `cargo about` and covers **only** the Rust dependency graph of `exaudfclient`. So the redistributed image bundles OS/runtime third-party components carrying attribution and (for GPL/LGPL components) source-offer obligations that are currently undocumented.

This is a **notice/attribution + source-offer compliance gap only** — there is no license conflict or copyleft contamination. All bundled components are unmodified upstream binaries shipped as separate files; glibc is LGPL-2.1, and libstdc++/libgcc carry the GCC Runtime Library Exception, so nothing propagates to `exaudfclient` or to user UDFs.

- **Goals** — Make the redistributed `lc-rs.tar.gz` compliant by shipping, at `/exaudf`, a `THIRD-PARTY-OS-LICENSES.md` manifest that maps every bundled OS/runtime component to its SPDX license, reproduces the canonical license texts (incl. the GCC Runtime Library Exception), and carries an inline three-year written source offer. The manifest is **generated** by `cargo about` from committed `dist/` boilerplate (so no verbatim license text is committed); it is git-ignored and produced before the Docker build. Bump the version to 0.13.1.
- **Non-Goals** — No CI staleness gate (the apk set changes rarely; drift is caught in review). No rootfs trimming or reduction of the bundled component set. No change to `Dockerfile.debian` (it produces no distribution tarball and is not referenced by any script, test, or CI — it is a dev/test image only). No API change.

### Decision

**Generate** the OS-license manifest with `cargo about`, driven by committed boilerplate under `dist/`: a synthetic dependency-free crate (`dist/os-attribution`) whose `license` expression enumerates the bundled OS/runtime licenses, an `about-os.toml` accepted list, a `os-licenses.hbs` template (apk→SPDX table + glibc/GCC section + three-year written offer + the license-text loop), and a `generate-os-licenses.sh` script. `cargo about` renders each license's text from its embedded SPDX store, and the script appends the GCC Runtime Library Exception text (which cargo-about does not emit for a `WITH` clause). The generated `dist/THIRD-PARTY-OS-LICENSES.md` embeds full texts and is therefore **git-ignored** (no verbatim license text committed). The generator runs before the Docker build — in `scripts/install.sh`, `scripts/ci-it-local.sh`, and the CI `build-slc` job — and `Dockerfile.alpine` COPYs the result into `/exaudf` in the `runtime` stage so the `packager` stage carries it into `lc-rs.tar.gz`. No CI staleness gate (apk/runtime composition changes only on an `apk add` edit or an `alpine:3`/builder bump, both review-visible).

#### Architecture

```
Repo root (committed)                 Dockerfile.alpine                 lc-rs.tar.gz
┌─────────────────────────┐           ┌──────────────────┐             ┌──────────────────────┐
│ THIRD-PARTY-OS-LICENSES │           │ runtime stage:   │             │ /exaudf/             │
│ licenses/ (SPDX texts)  │──COPY────▶│  COPY LICENSE     │──packager──▶│   LICENSE            │
│ LICENSE (existing)      │           │  THIRD-PARTY-*.md │   (tar)     │   THIRD-PARTY-*.md   │
│ THIRD-PARTY-LICENSES.md │           │  licenses/        │             │   THIRD-PARTY-OS-*.md│
└─────────────────────────┘           └──────────────────┘             │   licenses/          │
                                                                        └──────────────────────┘
```

#### Patterns

| Pattern | Where | Why |
|---------|-------|-----|
| Committed static notice bundle (no codegen) | `THIRD-PARTY-OS-LICENSES.md` + `licenses/` | OS/runtime component set is small and changes rarely; a generator + gate is unjustified overhead (per interview) |
| Notices added in the `runtime` stage, not `packager` | `Dockerfile.alpine` | The `packager` stage tars the runtime rootfs; placing files in `runtime` guarantees they land in `lc-rs.tar.gz` without touching the tar exclude list |
| Written source offer for GPL/LGPL, attribution-only for permissive | `THIRD-PARTY-OS-LICENSES.md` | GPLv2 §3 / LGPL-2.1 require source-or-offer; permissive (MIT/BSD/Apache/MPL/Zlib) require notice reproduction only |

### Consequences

| Decision | Alternatives Considered | Rationale |
|----------|------------------------|-----------|
| Commit manifest, no CI gate | Mirror the `cargo about` staleness gate for the OS layer | apk/runtime set changes rarely and only via explicit Dockerfile edits or base-image bumps, both review-visible; a gate is maintenance cost with little payoff (interview decision) |
| Written offer pointing at upstream source (Alpine aports + snapshot.debian.org) | Bundle the actual source tarballs in the image | GPLv2 §3(b)/LGPL-2.1 permit a 3-year written offer for unmodified upstream binaries; bundling source bloats the SLC for no practical benefit (sources are unmodified upstream) |
| Patch bump 0.13.0 → 0.13.1 | Fold into 0.13.0; minor bump 0.14.0 | v0.13.0 is being tagged/released independently right now; this is a notice-only change with no API surface, so a patch bump is correct (interview decision) |
| Cover Alpine image only | Also produce a Debian notice bundle | `Dockerfile.debian` produces no distribution tarball and is referenced by no script/test/CI; only `lc-rs.tar.gz` from `Dockerfile.alpine` is redistributed |

## Concrete Bill of Materials

Bundled in `lc-rs.tar.gz`, not covered by `THIRD-PARTY-LICENSES.md` (gathered by inspecting the built runtime image):

**Alpine apk packages** (from `/lib/apk/db/installed`, `P:`/`L:` fields):

| Package(s) | SPDX |
|------------|------|
| alpine-baselayout, alpine-baselayout-data, apk-tools, busybox, busybox-binsh, libapk, scanelf, ssl_client | GPL-2.0-only |
| alpine-keys, alpine-release, musl | MIT |
| musl-utils | MIT AND BSD-2-Clause AND GPL-2.0-or-later |
| ca-certificates, ca-certificates-bundle | MPL-2.0 AND MIT |
| libcrypto3, libssl3 (OpenSSL 3) | Apache-2.0 |
| zlib | Zlib |
| tzdata | Public-Domain |

**glibc / GCC runtime libs copied from `rust:1.92-bookworm` (Debian 12):**

| Library file(s) | SPDX |
|-----------------|------|
| `/lib/x86_64-linux-gnu/libc.so.6`, `libm.so.6`, `/lib64/ld-linux-x86-64.so.2`, `/usr/lib/x86_64-linux-gnu/libnss_files.so.2`, `libnss_dns.so.2` | LGPL-2.1-or-later |
| `/lib/x86_64-linux-gnu/libstdc++.so.6`, `libgcc_s.so.1` | GPL-3.0 WITH GCC-Runtime-Library-Exception |

Obligation tiers:
1. **Permissive** (MIT, BSD-2-Clause, Apache-2.0, MPL-2.0, Zlib, Public-Domain): reproduce copyright + license text. Apache-2.0 also preserves any NOTICE.
2. **GPL-2.0 / LGPL-2.1**: license text PLUS corresponding source or a 3-year written offer. The standard compliance posture for unmodified upstream binaries is a written offer pointing at upstream source (Alpine aports for the exact apk versions; Debian source packages / snapshot.debian.org for glibc & gcc-12 runtime). libstdc++/libgcc ship the GCC Runtime Library Exception text alongside GPL-3.0.

## Dependencies

- External license-mechanics research (cited in the ADR above): GPLv2 §3 written-offer mechanics ([FSF GPLv2 FAQ](https://www.gnu.org/licenses/old-licenses/gpl-2.0-faq.en.html), [SFLC Practical Guide to GPL Compliance](https://softwarefreedom.org/resources/2008/compliance-guide.tex)) and the GCC Runtime Library Exception ([FSF GCC Exception 3.1 FAQ](https://www.gnu.org/licenses/gcc-exception-3.1-faq.en.html), [exception text](https://www.gnu.org/licenses/gcc-exception-3.1.html)).
- Canonical SPDX license texts are obtained from the SPDX license-list data and committed verbatim under `licenses/`.

## Implementation Tasks

1. **Author the OS-layer notice content [expert]**
   - 1.1 Write `licenses/` canonical SPDX texts: `GPL-2.0-only.txt`, `LGPL-2.1.txt`, `Apache-2.0.txt`, `MPL-2.0.txt`, `MIT.txt`, `BSD-2-Clause.txt`, `Zlib.txt`, `GCC-exception-3.1.txt` (verbatim SPDX/FSF text). [expert]
   - 1.2 Write `THIRD-PARTY-OS-LICENSES.md`: the apk package → SPDX table, the copied-glibc/GCC-libs section with file paths, per-license pointers into `licenses/`, and the inline 3-year written source offer (Alpine aports for apk versions; Debian source packages / snapshot.debian.org for glibc & gcc-12 runtime), preserving any Apache-2.0 NOTICE. [expert]
2. **Wire the bundle into the distributed tarball**
   - 2.1 Edit `Dockerfile.alpine` `runtime` stage to `COPY THIRD-PARTY-OS-LICENSES.md /exaudf/` and `COPY licenses/ /exaudf/licenses/` alongside the existing `COPY LICENSE THIRD-PARTY-LICENSES.md /exaudf/`.
3. **Version bump**
   - 3.1 Bump `[workspace.package].version` `0.13.0` → `0.13.1` and the `exasol-udf-sdk` `[workspace.dependencies].version` `0.13.0` → `0.13.1` in root `Cargo.toml`; run `cargo check` to regenerate `Cargo.lock`. Leave `exarrow-rs` unchanged.
4. **Verification**
   - 4.1 Build the artifact tarball, extract it, and assert `exaudf/THIRD-PARTY-OS-LICENSES.md` + `exaudf/licenses/` (with all referenced texts) land at `/exaudf` alongside the existing `LICENSE` and `THIRD-PARTY-LICENSES.md`.
   - 4.2 Run the db-roundtrip integration suite against the rebuilt image to confirm no regression.

Explicitly excluded (per interview / scope): a CI staleness gate for the OS manifest, and any rootfs trimming.
